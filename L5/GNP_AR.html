Step 2. Look into the PACF to select the AR model order

acf(da$gnp.g,lag=12)

pacf(da$gnp.g,lag=12)

The ACF and PACF can be quite different.

At lag 3, ACF is positive, while PACF is negative. Controling for the impact of lag 1 and lag 2, the impact of lag 3 is negative, though the correlation of lag 3 and original data with out considering the lag 1 and lag 2 is positive.

Since the PACF is significant up to lag 3, we use the AR(3) model

To call the AR model with lag 3, use the function arima(data,order=c(3,0,0))

m1 = arima(da$gnp.g,order=c(3,0,0))

Step 3. Look into the IC to select the AR(p) model

tmp = arima(da$gnp.g,c(3,0,0))
AIC(tmp)
## [1] -1607.117
aic = NULL
bic = NULL
for (i in 0:12){
  tmp = arima(da$gnp.g,c(i,0,0))
  tmpaic = AIC(tmp)
  tmpbic = BIC(tmp)
  aic = append(aic,tmpaic)
  bic = append(bic,tmpbic)
}

Step 5. Modified AR(9) model

From the PACF, the coefficients are only significant at lag 1,2,3,9 but not significant at other lags. So restrict the coefficients at lag 4,5,6,7,8 to 0 in the AR(9) model.

The subcommand fixed= enables users to fix parameter values, where NA means estimation and 0 means fixing the parameter to 0. The ordering of the parameters can be found using gnp.g.ar9$coef.

const = rep(NA,10)
const[4:8]=0
gnp.g.ar9.const = arima(da$gnp.g, order = c(9, 0, 0), fixed = const)

The p value for the AR(9) model is greater than 10%. The model is definitely sufficient enough.

The p value for the AR(3) model is greater than 5%. Therefore, with 3 lags, the serial correlation problem is acceptable already.

The p value for the modified AR(0) is greater than 10%. With the lag at 9, the serial correlation is removed.
